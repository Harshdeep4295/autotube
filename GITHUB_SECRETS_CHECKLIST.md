### Step 2: Add YouTube Credentials

| Secret Name | Value | Get From | How to Get |
|-------------|-------|----------|-----------|
| `YOUTUBE_CLIENT_SECRETS` | Contents of client_secrets.json | GCP Console → APIs & Services → Credentials | Download OAuth 2.0 credentials (Desktop App) |
| `YOUTUBE_TOKEN_JSON` | Contents of data/youtube_token.json | Local file (generated by running `python get_youtube_token.py`) | Run OAuth flow once locally |

**How to get these:**

**YOUTUBE_CLIENT_SECRETS:**
```bash
# Download from GCP Console
# 1. Go: https://console.cloud.google.com/apis/credentials
# 2. Click "Create Credentials" → "OAuth 2.0 Client ID" → "Desktop Application"
# 3. Download the JSON file
# 4. Open and copy the entire JSON:

cat client_secrets.json | jq '.'
# Paste this entire output as YOUTUBE_CLIENT_SECRETS value
```

**YOUTUBE_TOKEN_JSON:**
```bash
# Generate locally (one-time)
python get_youtube_token.py
# Browser opens → Click "Allow" → Token saved to data/youtube_token.json

# Copy the token:
cat data/youtube_token.json | jq '.'
# Paste this entire output as YOUTUBE_TOKEN_JSON value
```

**Time: 10 minutes**

**Validation:**

## How to Populate Each Secret

### For JSON Secrets (client_secrets.json, service account key, etc.)

**Method 1: Copy from file directly**

```bash
# Display as single line (no line breaks)
cat autotube-veo-key.json | jq -c '.'
# Copy the output → Paste into GitHub secret field
```

**Method 2: Pretty-print then copy all**

```bash
cat autotube-veo-key.json
# Select all (Cmd+A on Mac, Ctrl+A on Linux/Windows)
# Copy (Cmd+C or Ctrl+C)
# Paste into GitHub secret field
```

**Method 3: Use clipboard (macOS)**

```bash
cat autotube-veo-key.json | pbcopy
# Now just paste (Cmd+V) into GitHub
```

### For API Keys (single-line strings)

```bash
# Print the key
echo $ANTHROPIC_API_KEY
# Or get from original source (console.anthropic.com, etc.)
# Copy and paste into GitHub
```

## How to Update a Secret

If you need to change a secret value (e.g., refresh YouTube token):

1. Go to **Settings → Secrets and variables → Actions → Secrets**
2. Click the secret name (e.g., `YOUTUBE_TOKEN_JSON`)
3. Click **Update** button
4. Paste new value
5. Click **Save changes**

**Note:** Old value is discarded. New value takes effect immediately on next workflow run.

---

## How to Delete a Secret

If you want to remove a secret:

1. Go to **Settings → Secrets and variables → Actions → Secrets**
2. Click the secret name
3. Click **Delete** (trash icon)
4. Confirm

**Note:** The secret is immediately deleted. Any workflow using it will fail until you add a new value.

## Troubleshooting

### "Secret not found" error in workflow

**Problem:** Workflow fails with "Secret XXXX not found"

**Cause:** Secret name typo or secret not added

**Fix:**
1. Check secret name is spelled exactly (case-sensitive)
2. Verify it appears in Settings → Secrets list
3. Re-add if it was deleted

### "Invalid JSON" error

**Problem:** GitHub rejects the secret with JSON error

**Cause:** JSON contains line breaks or is malformed

**Fix:**
```bash
# Use -c flag to compact JSON (remove line breaks)
cat autotube-veo-key.json | jq -c '.'
# Copy this one-liner into GitHub
```

### "Token expired" errors during workflow

**Problem:** Workflow fails with "invalid_grant: Token expired"

**Cause:** YouTube token in YOUTUBE_TOKEN_JSON has expired

**Fix:**
```bash
# Generate new token
python get_youtube_token.py

# Copy new token to GitHub
cat data/youtube_token.json | jq '.'
# Update GitHub Secret: YOUTUBE_TOKEN_JSON

# Re-run workflow
```

### "Permission denied" errors

**Problem:** Workflow fails with "Permission denied" or "403 Forbidden"

**Cause:** Service account key in `AI_VIDEO_GCP_SERVICE_ACCOUNT_JSON` is invalid or lacks permissions

**Fix:**
1. Verify the JSON is correct: `cat autotube-veo-key.json | jq '.project_id'`
2. Check service account has required IAM roles: `gcloud projects get-iam-policy`
3. Re-create key if needed: `gcloud iam service-accounts keys create autotube-veo-key.json --iam-account=...`
4. Update GitHub Secret with new key
