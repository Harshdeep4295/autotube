3. cloudbuild.yaml (3.8 KB)
   Location: /Users/harshdeepsingh/Projects/git_projects/autotube/cloudbuild.yaml
   Purpose: Automate Docker builds in Cloud Build
   Steps:
     1. Build Docker image (cache layers from previous builds)
     2. Push to Artifact Registry (tagged by git SHA + latest)
     3. Optional: Run dry-run tests
     4. Optional: Auto-deploy to Cloud Run
   Trigger: Manual (gcloud builds submit) or auto on git push to main
   Cost: 10-15 min per build, 120 min/day free

4. deploy-cloud-run.sh (7.3 KB) [EXECUTABLE]
   Location: /Users/harshdeepsingh/Projects/git_projects/autotube/deploy-cloud-run.sh
   Purpose: One-command deployment to Cloud Run
   Does:
     - Validates GCP project + required APIs enabled
     - Checks Docker image exists in Artifact Registry
     - Verifies all required secrets in Secret Manager
     - Deploys to Cloud Run with proper resource allocation
     - Shows service URL + next steps
   Usage: ./deploy-cloud-run.sh [--project --region --memory --cpu]
   Run this after: gcloud builds submit or docker push

5. manage-secrets.sh (8.1 KB) [EXECUTABLE]
   Location: /Users/harshdeepsingh/Projects/git_projects/autotube/manage-secrets.sh
   Purpose: Manage secrets in Google Secret Manager
   Commands:
     - create [--from-file FILE]  : Create secrets from .env
     - list                        : List all secrets
     - delete SECRET_NAME          : Delete a secret
     - grant [SERVICE_ACCOUNT]     : Grant Cloud Run access
     - verify                      : Check all required secrets exist
   Usage: ./manage-secrets.sh create
   Run this first: Before deployment

# 2. Build Docker image (5 min)

./deploy-cloud-run.sh

# 4. Monitor logs (real-time)
gcloud run logs read autotube-pipeline -f

# 5. Schedule runs (optional)

  --http-method=POST \
  --uri=<YOUR_CLOUD_RUN_URL>
